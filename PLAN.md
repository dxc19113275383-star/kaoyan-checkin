# PLAN.md — 项目计划（2026-07-02 立项）

> 本文档是功能规划的唯一权威来源，由 2026-07-02 的全面功能审计产生。
> 每完成一项就把状态改为 ✅ 并注日期。架构/部署真相见 `CLAUDE.md`（不重复）。

## 背景：审计结论摘要

- **打卡页与学习行为曾双轨脱节**（致命，已修）：打卡 `state.data`（手动勾选）与学习脊柱 `state.learn`（自动记录）互不通信，两套 streak 可互相矛盾，热力图/周报不吃学习行为。
- **SW 缓存策略吃更新**（阻断）：`sw.js` 对 `data/*.json` cache-first，不 bump 版本号用户看不到新内容。开发中已实测复现（改完代码刷新拿到的还是旧页）。
- **占位入口多**：CET4/CET6/专业英语词库（文件不存在）、logic 题库（空白）、写作模板（有 UI 无数据）、「今日学习/真题题库/AI模拟考」占位卡——"功能显得少而且破"的直接原因。
- **零宽恕机制**：漏一天 streak 清零、无补卡，与"易放弃用户"的产品哲学冲突。
- 模块闭环度：背单词 95% / 数学做题 90% / 错题中心 90% / AI 85% / 作文 80% / 课程 75% / 阅读 70%（无配套题）/ 长难句 65%（无复习机制）。
- 其余：换设备数据丢失只靠手动导出（云同步是既定红线，不做）；AI 报错文案技术性；`gen_*` 缓存无上限；iOS PWA 推送引导误导；812KB 词库加载无进度提示。

## 红线（不因本计划改变）

私人应用、无后端/账号/云同步；内容一律离线脚本生成 → 静态 `data/`；UI 极简禁 emoji；改动必回写 CLAUDE.md + CHANGELOG。

## P3 — 健壮与体验

- `gen_*` 缓存 LRU 上限；自建词库/题库大小警告
- 词库大文件加载进度提示
- iOS 下隐藏 Web Push 开关，只留应用内提醒
- 周报升级：本周背词数/做题数/正确率/学习分钟一屏（脊柱数据已有，只差呈现）

## "可日用上线"的定义

P0 全部完成 = 数据可信、无空白入口、内容更新可达 → 可放心让燕燕每天用。
P1.1 + 1.2 + 1.5 完成 = "功能少"观感消失。
